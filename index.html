⚡ Zero-config — connects itself

This dashboard finds and connects to your agent automatically — no URL to paste. It reconnects to a known agent on every visit and keeps watching for a local agent in the background. Just start the agent with the one command below; the dashboard does the rest.

Mac / Linux curl -fsSL https://vigil.kneuralabs.com/agent/run-agent.sh | bash
Windows irm https://vigil.kneuralabs.com/agent/run-agent.ps1 | iex

Want a permanent URL that survives restarts? Add --permanent --hostname agent.yourdomain.com — see agent/README.md.

Agent on another host? Set it manually (optional)
1
Run the agent on an intranet host
Copy the repo's agent/ folder to any box that can reach your services. Zero dependencies — just Python 3:
cp config.example.json config.json && python3 vigil_agent.py --config config.json
2
Expose its /events endpoint
Easiest — a Cloudflare Tunnel (no inbound ports; gives a public HTTPS URL). The agent already sends CORS headers:
cloudflared tunnel --url http://localhost:8787
Alternative: set webhook_url in the config to PUSH events to a relay you control, then use that relay's read URL.
3
Paste the URL below & Connect (only needed for a remote tunnel)
A local agent is detected automatically. For a remote tunnel, paste its /events URL once — the dashboard remembers it and reconnects on every future visit. It polls every 10s, validates the JSON, de-duplicates, and streams only new events into the feed.
